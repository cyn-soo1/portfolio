Graphic Design · Course Project

Visual Design Practice

Posters, campaigns, advertisements & brand identity — built over 3 months of structured study

The Goal

Learn professional design tools and visual communication principles through real briefs — producing work that functions the way actual client work does, with constraints around layout, typography, and message.

My Process

Each project started with understanding the brief and audience. I then explored color palettes and type pairings in Illustrator before moving into composition in Photoshop. Iteration and feedback from the course shaped the final pieces.

What I Made

Sports event posters, social media thumbnails, business card systems, a public awareness campaign ("Break the Silence"), a product advertisement (Toyota Yaris), and several artist/brand identity edits.

What I Learned

Color hierarchy matters more than anything — the eye goes where you direct it. Typography is invisible when it works and loud when it doesn't. Constraints (a real brief, a real format) make you better faster than total creative freedom.

02
Software Development · Personal Project

Contact Manager App

A dark-mode desktop application with secure login and persistent data storage

The Goal

Build a functional desktop application from scratch — something polished enough to actually use, with a login system, data that saves between sessions, and a UI that doesn't look like a school project.

My Process

I planned the features first: what does a contact manager actually need? Then built it incrementally — login system first, then the contact list, then data persistence via JSON, then styling with CustomTkinter. Used AI assistance to understand Python patterns I hadn't encountered yet.

What I Built

A fully working app with encrypted local login, add/edit/delete contacts, automatic JSON saving, a calendar integration, and a dark-mode UI. Released on GitHub with a README, license, and packaged installer.

What I Learned

Shipping something — actually finishing it, packaging it, writing the README — is a completely different skill from just coding it. The UI decisions (dark mode, layout, spacing) were as important as the functionality.

Character Design · Brand Identity · Ongoing

CynSoo — VTuber Persona & Visual Identity

Designing a consistent original character across 3D model, illustration, and streaming presence

The Goal

Create a VTuber persona with a strong, recognisable visual identity — one that works across a 3D model, commissioned artwork, stream overlays, and social media, while staying consistent and true to a clear character concept.

The Design Decisions

Every visual choice was intentional. The character needed to feel like an extension of a real aesthetic — not just a random anime girl, but a specific person with a specific vibe.

What Exists So Far

A working VRoid 3D model (VSeeFace-ready), two commissioned character art pieces in different styles, a demon/angel variant concept, stream banner backgrounds, and a defined color palette used across all assets.

What This Taught Me

Consistency is a design problem. Keeping a character recognisable across a 3D model, a flat illustration, and a different artist's style requires defining the non-negotiables — the things that make her her — before anything else.

Core Design Decisions

  • Heterochromia (red + purple eyes) — deliberate asymmetry to make the face immediately memorable and hint at a dual nature
  • Deep purple + crimson red palette — matches the astral/cosmic brand theme; dark enough to feel serious, warm enough to feel expressive
  • Space buns as signature silhouette — readable at small sizes (profile pictures, stream thumbnails) even when other details compress
  • Purple hoodie as default outfit — casual and approachable vs. the cyberpunk/demon variants, creates clear "mode" switching
  • Demon variant with feathers + wings — alternate persona for different content moods; same palette, different energy
